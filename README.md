# XFEL Vacuum Birefringence — X-ray Transmission Simulation (Geant4)

This Geant4 simulation calculates the transmission of primary polarized X-ray photons (**8.766 keV**) through a configurable Kapton window followed by an Air gap. It records the position, momentum, energy, and particle type of **every** particle (γ, e⁻, e⁺) that physically crosses the downstream virtual screen detector.

The simulation is fully multithreaded and uses polarized EM physics (Polarized Compton, Photoelectric, Bremsstrahlung, etc.) to correctly model X-ray–matter interactions.

## Physics

### Processes Registered

| Particle | Processes |
| :--- | :--- |
| **γ (Gamma)** | Polarized Photoelectric Effect, Polarized Compton Scattering, Polarized Gamma Conversion |
| **e⁻ (Electron)** | Multiple Scattering, Polarized Ionisation, Polarized Bremsstrahlung |
| **e⁺ (Positron)** | Multiple Scattering, Polarized Ionisation, Polarized Bremsstrahlung, Polarized Annihilation |

### Geometry (default)

| Volume | Material | Transverse Size | Longitudinal |
| :--- | :--- | :--- | :--- |
| World | Air | — | — |
| Kapton Window | Kapton | 1 m × 1 m | configurable (default 50 µm) |
| Air Gap | Air | 1 m × 1 m | configurable (default 1 m) |
| Virtual Screen | Air | 1 m × 1 m | 1 µm (infinitesimally thin) |

The screen has the same material as the Air gap (Air) to avoid any additional interactions; it acts as a pure recording plane.

## Command Line Parameters

### Python Automation Script (`RunAndAnalyze.py`)

Handles compiling, macro generation, simulation execution, and analysis in a single command.

```bash
python RunAndAnalyze.py [options]
```

| Option | Type | Default | Description |
| :--- | :--- | :--- | :--- |
| `--kapton <val>` | float | `50.0` | Kapton window thickness in **µm** |
| `--air <val>` | float | `1.0` | Air gap thickness in **m** |
| `--threads <val>` | int | `5` | Number of Geant4 worker threads |
| `--primaries <val>` | float | `2.1e9` | Number of primary γ particles (supports scientific notation) |
| `--env <name>` | str | `geant4` | Conda/Micromamba environment name |
| `--output-dir <path>` | str | auto | Custom output directory (default: `results/results_<k>um_<a>m/`) |

**Example:**
```bash
python RunAndAnalyze.py --kapton 50 --air 1.0 --threads 8 --primaries 1e6
```

### Geant4 Macro UI Commands

These commands can be placed in `.mac` files before `/run/initialize`:

```
/testem/det/setKaptonThickness 50.0 um
/testem/det/setAirThickness    1.0 m
/testem/det/update
```

## Installation and Setup

### Prerequisites

- Geant4 (≥ 10.7, built with multithreading and ROOT support)
- CMake ≥ 3.10
- C++17 compiler (GCC or Clang)
- Python 3 with: `uproot`, `numpy`, `matplotlib`

### Create the Conda/Micromamba Environment

```bash
# Using Micromamba (recommended)
micromamba env create -f environment.yml
micromamba activate geant4

# Using Conda
conda env create -f environment.yml
conda activate geant4
```

### Option A — Full Automation (Recommended)

```bash
python RunAndAnalyze.py --kapton 50 --air 1.0 --threads 8 --primaries 1e6
```

Results are saved automatically to `results/results_50.0um_1.0m/`.

### Option B — Manual Compilation and Execution

```bash
# 1. Configure and build
mkdir -p build && cd build
cmake ..
make -j$(nproc)

# 2. Run the simulation (macro file, number of threads)
./XFELVB xray.mac 8
```

## Output Files

### `results.root`

The raw Geant4 output ROOT file (written to the `build/` directory). Contains:
- **`Gamma` TTree** — one row per particle hit on the screen (all types: γ, e⁻, e⁺)
- **`Summary` TTree** — per-thread primary count and gamma hit count
- Several 1D and 2D histograms for angular distributions and polarization

### `*.npz` — Compressed Hit Arrays

The `.npz` file is a NumPy compressed archive containing **all particles** that hit the screen. Each key is a 1D NumPy array where index `i` corresponds to the `i`-th hit.

#### Array Keys

| Key | Unit | Description |
| :--- | :--- | :--- |
| `x` | µm | X-coordinate of the hit on the screen plane |
| `y` | µm | Y-coordinate of the hit on the screen plane |
| `e` | keV | Kinetic energy of the particle at the moment of screen crossing |
| `particle` | integer (dimensionless) | Particle type code (see table below) |

#### Particle Type Codes

| Code | Particle |
| :--- | :--- |
| `1` | γ — Gamma / X-ray photon |
| `5` | e⁻ — Electron |
| `9` | e⁺ — Positron |

#### Loading and Filtering in Python

```python
import numpy as np

# Load the file
data = np.load("xray_Primaries1.0e+06_50.0umKapton_1.0mAir.npz")

x        = data['x']        # hit x-position [µm]
y        = data['y']        # hit y-position [µm]
energy   = data['e']        # kinetic energy [keV]
particle = data['particle'] # particle type code

# --- Filter by particle type ---
gamma_mask    = particle == 1   # X-ray photons / gammas
electron_mask = particle == 5   # electrons
positron_mask = particle == 9   # positrons

# Example: work only with gammas
x_gamma = x[gamma_mask]
y_gamma = y[gamma_mask]
e_gamma = energy[gamma_mask]

print(f"Total screen hits : {len(x):,}")
print(f"  γ hits          : {np.sum(gamma_mask):,}")
print(f"  e⁻ hits         : {np.sum(electron_mask):,}")
print(f"  e⁺ hits         : {np.sum(positron_mask):,}")
print(f"γ transmission    : {np.sum(gamma_mask) / 1e6 * 100:.4f} %")
```

#### Example Plot

```python
import matplotlib.pyplot as plt
from matplotlib.colors import LogNorm

# 2D spatial map — gammas only
fig, axes = plt.subplots(1, 2, figsize=(14, 5))

axes[0].hist2d(x_gamma, y_gamma, bins=100,
               range=[[-300, 300], [-300, 300]],
               cmap='viridis', norm=LogNorm())
axes[0].set_xlabel("x [µm]")
axes[0].set_ylabel("y [µm]")
axes[0].set_title("γ Screen Hits")

# Energy spectrum — all types
axes[1].hist(energy[gamma_mask],    bins=120, range=(0, 12),
             histtype='step', label='γ',  color='royalblue')
axes[1].hist(energy[electron_mask], bins=120, range=(0, 12),
             histtype='step', label='e⁻', color='tomato')
axes[1].hist(energy[positron_mask], bins=120, range=(0, 12),
             histtype='step', label='e⁺', color='mediumseagreen')
axes[1].set_yscale('log')
axes[1].set_xlabel("Energy [keV]")
axes[1].set_ylabel("Counts")
axes[1].legend()

plt.tight_layout()
plt.show()
```

## Notes

- The **virtual screen** is made of Air (same material as the gap volume) so it does not introduce any additional scattering or absorption.
- All particles hitting the screen are recorded **without prescaling** — every crossing is saved.
- Multithreading uses Geant4's built-in ntuple merging (`SetNtupleMerging(true)`). The master thread performs the final `Write()` + `CloseFile()` after all workers complete.
- The γ **transmission rate** reported in `summary.md` is `N_gamma_hits / N_primaries × 100 %`.
